# مدیریت سفارش رستوران

## موضوع
پیاده‌سازی Web API برای مدیریت سفارش‌های رستوران.

## الزامات

### عملیات CRUD
1. **ایجاد سفارش** — ثبت سفارش جدید
2. **لیست سفارش‌ها** — دریافت همه سفارش‌ها
3. **دریافت سفارش** — دریافت یک سفارش با شناسه
4. **به‌روزرسانی وضعیت** — تغییر وضعیت سف order

### وضعیت‌های سفارش
- `Pending` — در انتظار
- `Preparing` — در حال آماده‌سازی
- `Ready` — آماده تحویل
- `Delivered` — تحویل شده

### معماری
- الگوی **Controllers / Services / Repositories**
- کد تمیز و جداسازی مسئولیت‌ها
- پروژه **.NET 9**

## خروجی مورد انتظار
- Web API قابل build
- endpointهای REST برای تمام عملیات

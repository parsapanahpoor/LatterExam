## شرح مسئله
یک Minimal API با .NET 9 ایجاد شده که نیاز به پیاده‌سازی مکانیزم احراز هویت ساده دارد. کاربر باید بتواند با ارسال نام کاربری و رمز عبور معتبر، یک **توکن** دریافت کند و سپس برای دسترسی به endpointهای محافظت‌شده، این توکن را ارسال نماید.

در کد اولیه، الزامات امنیتی رعایت نشده و هر کاربری بدون احراز هویت می‌تواند به منابع محافظت‌شده دسترسی پیدا کند.

## چالش‌های اصلی
- افزودن احراز هویت به endpointهای محافظت‌شده
- بررسی هدر `Authorization` با مقدار صحیح
- در صورت نبود یا نادرست بودن هدر، محدود کردن دسترسی و بازگرداندن خطای مناسب (**401**)
- استخراج و بررسی صحیح توکن Bearer

## الزامات
- `POST /login` — دریافت username/password، بازگشت token
- endpointهای محافظت‌شده (مثل `/protected`, `/profile`)
- هدر: `Authorization: Bearer <token>`
- پاسخ **401 Unauthorized** در صورت token نامعتبر یا نبودن

# چالش مقیاس‌پذیری در Minimal API با .NET 9

## اطلاعات آزمون
- **دسته‌بندی:** Concurrency-Scalability
- **نوع پروژه:** Minimal API (.NET 9)
- **مدت آزمون:** حدود ۶۰ دقیقه

## آدرس ریپازیتوری
- **URL:** https://git.arzyabit.ir/dotin/Dotnet9ScalabilityChallenge1071
- **Branch:** `main`

## شرح مسئله
یک Minimal API ساده با .NET 9 پیاده‌سازی شده که endpoint ای به نام `/process` برای پردازش همزمان درخواست‌ها دارد. هدف، رفع مشکل کاهش شدید عملکرد هنگام افزایش ناگهانی تعداد درخواست‌های همزمان است.

در کد اولیه، endpoint مذکور ۱۰٬۰۰۰ بار متد `SimulateWork` را **به‌صورت متوالی** با `await` داخل حلقه `for` اجرا می‌کند. هر فراخوانی حدود ۲ میلی‌ثانیه طول می‌کشد و در مجموع زمان پاسخ به حدود **۲۰ ثانیه** می‌رسد.

## چالش‌های اصلی
- شناسایی گلوگاه اجرای متوالی در حلقه `await`
- بهبود مقیاس‌پذیری با اجرای موازی وظایف
- حفظ عملکرد مناسب بدون استفاده از cache یا حافظه موقت

## الزامات و محدودیت‌ها
- فریم‌ورک: **.NET 9**
- ساختار SDK-style
- **بدون cache** و بدون حافظه موقت
- فقط پکیج‌های پایه .NET
- نیازی به فایل solution نیست

## راهنمای شروع
1. پروژه را اجرا کنید: `dotnet run`
2. درخواست POST به `/process` ارسال کنید (Postman یا curl)
3. زمان پاسخ را مشاهده کنید
4. گلوگاه را شناسایی و با اجرای موازی رفع کنید

```bash
curl -X POST http://localhost:5000/process
```
